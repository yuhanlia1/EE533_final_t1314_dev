dropfifo — Output Signals per State
State fifo_full pkt_ready write_ready proc_active Key Actions
S_IDLE 0 0 1 0 Reset write_count, read_issue_count; allow dbg read
S_FILL 0 0 1 0 Increment write_count each fill_write; latch pkt_len on lastword
S_PROCESS 1 1 0 1 CPU R/W BRAM via proc_mem; latch proc_done → proc_done_seen
S_DRAIN 1 0 0 0 Issue reads: read_issue_count++ ; push resp_fifo; track flight_count
start_write = (state==S_IDLE) && fifowrite && firstword
fill_write = (state==S_FILL) && fifowrite
proc_done_safe = proc_done_seen && !proc_rd_valid_d && !proc_read_req
drain_done = read_issue_count==pkt_len && flight_count==0 && resp_empty
convertible_fifo.v — hold_valid Register FSM
accept_in = in_wr && in_rdy
can_flush_hold = hold_valid && dropfifo_write_ready
in_rdy = !fifo_full && (!hold_valid || can_flush_hold)
incoming_first = (in_ctrl != 0)  [marks header word]
pipeline_datapath.v — boot_wait_r & CPU Activation
cpu_rst_local = rst | ~proc_active
en_reg = proc_active && !mem_stall && !cpu_boot_wait
proc_done driven by mm_stage when all instructions retire
Pipeline enabled only when: proc_active=1 && boot_wait_r=0 && !mem_stall
